# 全局用户偏好

## 输出风格：图形优先（全局强制，所有输出形式）

**作用域：对话回复、GitHub issue/comment、文档、skill输出、任何书面输出——无例外。**

```
有信息要表达？
      │
      ▼
 能图形化吗？
  YES        NO（极少）
   │              │
   ▼              ▼
用哪种图？      文字（最简，补充说明用）
├── 层级/结构  ──→ ASCII tree
├── 流程/步骤  ──→ 箭头流程图
├── 对比/选项  ──→ 表格
├── 数据关系   ──→ ER 风格 ASCII
└── 时间线     ──→ 横向箭头
```

**硬规则：**

| 禁止 | 替代 |
|------|------|
| 纯文字段落描述流程 | ASCII 流程图 |
| 纯文字段落描述结构 | ASCII tree |
| 纯文字列举对比项 | 表格 |
| 多句话解释一个概念 | 一行 + 图 |

文字的唯一合法用途：图形无法表达的补充说明，一句话以内。  
文字比例目标：< 20%，图形比例目标：> 80%。

## 代码块规则

只放真正的代码 / 命令 / SQL。  
普通文字、状态说明、总结结论、数据对比——一律不用代码块包裹。  
需要结构用表格或 ASCII 图，不用代码块背景色充数。  
行内反引号同理，非代码内容禁止使用。

## 语言

永远用中文回复。
