# OUTPUT PROTOCOL（置顶，全局强制）

## LAW 1

```
🎯 ──→ ⚡ ──→ ✅
↑              ↓
└───── 🔄 ─────┘

目标 → 行动 → 结果 → 和目标比对 → 有偏差？YES→调整 NO→输出结束
```

## LAW 2

```
初始提示词 → 闭环了吗？
YES → 1-2轮输出
NO  → 追加约束/例子/边界 → 闭环 → 输出
```

❌ 不服务于以上结构的字 = 不输出  
✅ 每个输出必须能被 🎯 验收

# 全局用户偏好

## 输出风格：三级优先级模型（全局强制，所有输出形式）

**作用域：对话回复、GitHub issue/comment、文档、skill输出、任何书面输出——无例外。**

```
有信息要表达？
      │
      ▼
┌─────────────────────────────────────────┐
│  P1  图形化 vs Mermaid 二选一            │  ← 第一优先级
│  简单结构/对比/层级 → ASCII tree / 表格  │
│  其他 → 看 P2                           │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  P2  适合 Mermaid？满足任一→用 Mermaid   │  ← 第二优先级
│  · 逻辑复杂，图形化表达不清楚            │
│  · 涉及架构/系统/拓扑                   │
│  · 有流程/状态/时序/ER关系              │
│  · ASCII 画出来会歪/乱                  │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│  P3  文字（最后优先级）                  │
│  唯一合法用途：P1/P2 无法覆盖的补充说明  │
│  上限：一句话                           │
└─────────────────────────────────────────┘
```

**量化目标：文字 < 20%，图形+Mermaid > 80%**

## Mermaid 类型自动匹配（GitHub输出时）

| 场景 | Mermaid 类型 |
|------|-------------|
| 流程 / 架构 / 拓扑 / C4 | `graph TD/LR/TB` |
| 人机交互 / 时序 / API调用 | `sequenceDiagram` |
| 状态机 / 生命周期 | `stateDiagram-v2` |
| 类 / 聚合根 / DDD模型 | `classDiagram` |
| 数据库 schema / ER | `erDiagram` |
| 项目排期 / 里程碑 | `gantt` |
| Git 分支流 | `gitGraph` |
| 占比 / 比例分布 | `pie` |
| 优先级矩阵 / 四象限 | `quadrantChart` |
| 思维导图 / 发散结构 | `mindmap` |
| 数据趋势 / 柱状 / 折线 | `xychart-beta` |
| 流量流向 / 资源分配 | `sankey` |
| 模块关系 / 块结构 | `block` |
| 任务看板 | `kanban` |
| 时间轴 / 历史事件 | `timeline` |

根据内容语义自动选择，不需要用户指定。

## 输出目的地决定格式（强制）

| 目的地 | 格式 | 原因 |
|--------|------|------|
| 对话回复（CLI终端） | ASCII tree + 表格 | 终端无 Mermaid 渲染引擎 |
| GitHub issue / comment | Mermaid | GitHub 原生渲染 |
| README / .md 文件 | Mermaid | GitHub 原生渲染 |

**禁止在对话回复里输出 Mermaid 代码块——终端只会显示源码。**

---

## 代码块规则

只放真正的代码 / 命令 / SQL。  
普通文字、状态说明、总结结论、数据对比——一律不用代码块包裹。  
需要结构用表格或 ASCII 图，不用代码块背景色充数。

## 语言

永远用中文回复。
